HTML 语法详解：从 Markdown 到 HTML

HTML 和 Markdown 一样，都是标签语言

快速理解：

HTML 就像 JSON 或 Markdown。JSON 是给程序看的结构化数据，HTML 是给浏览器看的结构化文档。Markdown 用 # 表示标题，HTML 用 <h1>；Markdown 用 - 表示列表，HTML 用 <ul><li>。概念完全一样，只是写法不同。你不需要记住每个标签怎么写，遇到不会的查一下或者问 AI 就行。

1. 标题

Markdown 用 # 数量表示标题层级，HTML 用 <h1> 到 <h6> 标签。一个页面通常只有一个 h1，其他用 h2、h3 组织层级。

Markdown
# 一级标题 ## 二级标题 ### 三级标题
HTML
<h1>一级标题</h1> <h2>二级标题</h2> <h3>三级标题</h3>
渲染结果

一级标题

二级标题

三级标题

2. 段落

Markdown 里空行分隔段落，HTML 需要用 <p> 标签显式包裹每个段落。这是 HTML 比 Markdown "啰嗦"的地方，但也更明确。

Markdown
这是第一段。 这是第二段。 这是第三段。
HTML
<p>这是第一段。</p> <p>这是第二段。</p> <p>这是第三段。</p>
渲染结果

这是第一段。

这是第二段。

这是第三段。

3. 换行

Markdown 段落用空行分隔。同一段内想换行有两种写法：行末加 两个空格再回车，或直接插入 <br> 标签。

HTML 不一样。浏览器把源码里的所有空白（空格、回车、连续空行）折叠成一个空格，源码敲多少回车都没用。要换行必须用 <br> 自闭合标签，没有第二种写法。

Markdown
第一行 第二行 第三行 上一段末尾空两格 + 回车 否则会被当成同一段
HTML
<!-- 不换行：源码回车被忽略 --> <p>第一行 第二行 第三行</p> <!-- 换行：用 br 标签 --> <p>第一行<br> 第二行<br> 第三行</p>
渲染结果对比

第一行 第二行 第三行（源码回车，显示成一行）

第一行
第二行
第三行（用 br 真正换行）

4. 粗体

Markdown 用双星号 **文字**，HTML 用 <strong> 标签。<strong> 不只是视觉加粗，还语义上表示"这段内容很重要"。

Markdown
这是 **粗体** 文字。
HTML
<p>这是 <strong>粗体</strong> 文字。</p>
渲染结果

这是 粗体 文字。

5. 斜体

Markdown 用单星号 *文字*，HTML 用 <em> 标签。<em> 表示"强调"，浏览器默认渲染为斜体。

Markdown
这是 *斜体* 文字。
HTML
<p>这是 <em>斜体</em> 文字。</p>
渲染结果

这是 斜体 文字。

6. 无序列表

Markdown 用 - 或 * 开头，HTML 用 <ul> 包裹 <li>。HTML 的结构更清晰：外层是列表容器，内层是列表项。

Markdown
- 第一项 - 第二项 - 第三项
HTML
<ul> <li>第一项</li> <li>第二项</li> <li>第三项</li> </ul>
渲染结果
  • 第一项
  • 第二项
  • 第三项

7. 有序列表

Markdown 用数字加点 1.，HTML 用 <ol> 包裹 <li>。浏览器会自动给 <ol> 里的项加上序号。

Markdown
1. 第一步 2. 第二步 3. 第三步
HTML
<ol> <li>第一步</li> <li>第二步</li> <li>第三步</li> </ol>
渲染结果
  1. 第一步
  2. 第二步
  3. 第三步

8. 链接

Markdown 用 [文字](URL)，HTML 用 <a href="URL">文字</a>。href 是属性，指定链接目标。

Markdown
[访问百度](https://www.baidu.com)
HTML
<a href="https://www.baidu.com">访问百度</a>
渲染结果

访问百度

9. 图片

Markdown 用 ![alt](src)，HTML 用 <img src="..." alt="...">。注意 <img> 是自闭合标签，不需要结束标签。alt 属性对无障碍访问很重要。

Markdown
![图片描述](https://via.placeholder.com/150)
HTML
<img src="https://via.placeholder.com/150" alt="图片描述">
渲染结果

10. 引用

Markdown 用 > 开头，HTML 用 <blockquote> 标签。引用块通常会有视觉上的缩进或左边框。

Markdown
> 这是一段引用文字。 > 可以有多行。
HTML
<blockquote> 这是一段引用文字。 可以有多行。 </blockquote>
渲染结果
这是一段引用文字。 可以有多行。

11. 行内代码

Markdown 用反引号 `code`，HTML 用 <code> 标签。行内代码通常会有等宽字体和浅色背景。

Markdown
使用 `console.log()` 调试。
HTML
<p>使用 <code>console.log()</code> 调试。</p>
渲染结果

使用 console.log() 调试。

12. 代码块

Markdown 用三个反引号包裹多行代码，HTML 用 <pre><code>...</code></pre>。<pre> 保留空格和换行，<code> 表示这是代码内容。

Markdown
``` function hello() { console.log("你好"); } ```
HTML
<pre><code>function hello() { console.log("你好"); }</code></pre>
渲染结果 
function hello() {
  console.log("你好");
}

13. 分隔线

Markdown 用 ---，HTML 用 <hr> 标签。<hr> 是自闭合标签，表示主题分隔。

Markdown
上面的内容 --- 下面的内容
HTML
<p>上面的内容</p> <hr> <p>下面的内容</p>
渲染结果

上面的内容

下面的内容

14. 表格

Markdown 表格语法简单但功能有限，HTML 表格用 <table>、<tr>（行）、<th>（表头）、<td>（单元格）组合，更灵活。

Markdown
| 姓名 | 角色 | |------|------| | 张三 | 管理员 | | 李四 | 编辑者 |
HTML
<table> <tr> <th>姓名</th> <th>角色</th> </tr> <tr> <td>张三</td> <td>管理员</td> </tr> <tr> <td>李四</td> <td>编辑者</td> </tr> </table>
渲染结果
姓名 角色
张三 管理员
李四 编辑者

15. 表单

Markdown 没有表单概念，这是 HTML 独有的交互元素。表单用 <form> 包裹，内部有 <label>（标签）、<input>（输入框）、<button>（按钮）等。

Markdown
（Markdown 不支持表单）
HTML
<form> <label for="name">姓名</label> <input type="text" id="name"> <button type="submit">提交</button> </form>
渲染结果

总结

HTML 比 Markdown "啰嗦"，但每个标签都有明确的语义。后端同学学 HTML，重点不是记住所有标签，而是理解：

  • 结构优先：先想清楚页面有哪些区块，再选标签
  • 语义重要：用 <strong> 而不是 <b>，用 <em> 而不是 <i>
  • 层级清楚：h1 → h2 → h3 不要跳级
  • 闭合规范：标签必须正确闭合，属性用引号包裹

掌握这些原则，你就能写出结构清晰、语义明确的 HTML，为后续学习 CSS 和 JavaScript 打好基础。
