HTML 对后端同学来说，到底是什么？

HTML 是给浏览器看的结构化文档，别当成难记的标签语法。你写的是“页面的骨架说明书”。

分类：HTML 难度：Beginner 预计阅读：12 min 产物：静态文章页

为什么后端同学一上来就容易把 HTML 学偏？

因为你会自然把它看成“前端的语言语法”，于是关注点落在标签名和闭合细节上。但更重要的问题其实是：页面的层级有没有清楚、语义是不是合理、浏览器能不能据此理解内容结构。

后端类比：

把 HTML 理解成接口返回的结构化数据。JSON 是给程序看的结构，HTML 是给浏览器看的结构。

Markdown 和 HTML 的真正区别

Markdown 解决的是“把内容快速写出来”，HTML 解决的是“把页面结构明确描述出来”。后者更适合被浏览器、样式系统、无障碍工具和 AI 一起消费。

Markdown
# 用户资料 姓名：张三 角色：管理员
HTML
<section> <h1>用户资料</h1> <p>姓名：张三</p> <p>角色：管理员</p> </section>

先掌握这些标签就够了

  • header / main / section：页面的大区块与语义边界
  • h1 ~ h3 / p / ul / li：内容层级与信息组织
  • button / a：动作与跳转的区别
  • form / input / label：用户输入结构
  • img：媒体内容占位

本节练习：把 Markdown 博客改成语义清晰的 HTML 页面

目标是把原本只有标题、正文、列表的 Markdown 内容，拆成有清楚区块的页面。AI 能不能继续帮你写样式，取决于这一层有没有先弄清楚。

最终产物：

一个纯 HTML 的个人文章页，具有 header、main、section、footer 的完整语义骨架。

直接可用的 AI 提示词

请把下面的 Markdown 内容改写成语义清晰的 HTML。 要求： 1. 使用 header、main、section 等语义标签 2. 不要写 CSS 3. 保持内容层级清楚 4. 给关键区域添加 class，方便后续写样式
