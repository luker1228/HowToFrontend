CSS 控制页面长什么样。

如果 HTML 是结构化响应，CSS 就像展示层配置。它决定字号、间距、边框、布局和 hover 状态，决定“浏览器看起来像不像产品”。

分类：CSS 难度：Beginner 预计阅读：14 min 产物：静态网页

先建立“样式旋钮”思维

后端同学常把 CSS 看成杂乱的属性列表。更有效的方法是把它看成一组旋钮：文字大小、区块间距、边框强弱、布局方向、状态反馈。页面丑，通常是这些旋钮没有被系统地拧过。

第一批必须会的属性

  • color / background / border：文字和区块的视觉基础
  • font-size / line-height / font-weight：可读性和层级
  • margin / padding：外部距离与内部留白
  • display / flex / grid：模块如何排
  • hover / transition：最轻量的交互反馈

布局的本质：让模块站对位置

大多数业务页面不需要复杂动画，但一定需要清楚布局。你可以先从 Flex 排一维，再用 Grid 处理卡片区和后台模块。

Flex 适合
导航栏 按钮组 表单行 卡片头部
Grid 适合
首页章节卡片 后台仪表盘 多列知识卡 Prompt Library

响应式是重排页面，不是缩小页面

移动端要重新决定内容优先级、按钮宽度和信息折行方式，单纯把桌面卡片缩窄是不够的。后端同学和 AI 协作时，要明确写出移动端规则。

本节练习：

给 HTML 第一课的文章页加样式，让它从“结构文档”变成“像样网页”，并补一个移动端单列布局。

动手 1：拧字号，看层级

只改下面这一行里的 font-size，比如把 16px 试到 55px，看右边标题在卡片里的存在感怎么变。字号是建立层级最直接的旋钮。

CSS 
body {
  font-family: system-ui, sans-serif;
  margin: 24px;
  background: #f5f4ee;
  color: #1a1a1a;
}
.card {
  padding: 16px;
  border: 1.5px solid #1a1a1a;
  border-radius: 12px;
  background: #fff;
  max-width: 360px;
}
.card-meta { margin: 0 0 12px; color: #6b6b6b; }
.card-body { margin: 0; line-height: 1.6; }
Preview

动手 2：拧 padding，看留白

改 .card 的 padding，从 8px 往上调。留白决定内容是挤在一起还是呼吸顺畅，这往往比加边框更影响“像不像产品”。

动手 3：拧 flex-direction，看方向

把 .card-actions 的 flex-direction 从 row 改成 column。一个属性就决定模块是横排还是竖排，这是布局的最小开关。

动手 4：拧 :hover，看反馈

在 CSS 最后加一条 .card-button:hover { ... }，改背景或颜色。把鼠标移到按钮上——状态反馈就是这点东西，缺了页面就“死”。

动手 5：拧 @media，看响应式

下面这版 CSS 在窄屏下卡片会挤成一坨。在末尾加一条 @media (max-width: 480px)，让 .card-grid 变成 grid-template-columns: 1fr。然后把预览窗口拉窄，看它重排。
