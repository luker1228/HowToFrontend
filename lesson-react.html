React 不是新语言，是组织前端代码的方法。

当页面从一个文件长成很多模块时，React 的价值就出来了。它帮你把 UI 看成组件，把组件看成函数，把页面看成状态驱动的结果。

分类：React 难度：Beginner 预计阅读：15 min 产物：React Todo 页面

先别背 API，先建立心智模型

React 最核心的事情只有三件：把页面拆成组件、让数据通过 props 流动、让局部状态通过 state 更新。理解这三件，你再看 useState 就不会像背咒语。

后端同学需要先搞懂的词汇表

  • npm / package.json：依赖管理与项目描述
  • Vite / Next：开发与构建框架
  • JSX：把 UI 结构写进 JavaScript 的方式
  • props：组件参数
  • state：组件内部会变化的数据
  • useState：最基础的状态钩子

组件拆分不是切文件，而是切职责

如果一个页面同时负责展示标题、处理输入、渲染列表、弹出详情、控制加载状态，它就已经包含多个职责了。React 的第一步是按职责分块，而不是按视觉区块乱切。

Todo Page
TodoPage ├─ TodoForm ├─ TodoList └─ EmptyState
后台页
DashboardPage ├─ Navbar ├─ Sidebar ├─ UserTable └─ UserDrawer

练习：把原生 JS Todo 改写成 React 组件

你会直观看到两个变化：事件处理不再散在 DOM 查询里，状态也不再手工拼接 HTML，而是通过数据映射直接生成 UI。

直接可用的 React 提示词

请帮我把一个原生 JavaScript Todo 页面改写成 React 版本。 要求： 1. 组件拆分为 TodoPage、TodoForm、TodoList、EmptyState 2. 使用 useState 管理任务列表和输入框值 3. 支持添加和删除任务 4. 当列表为空时显示空状态 5. 代码结构清楚，适合初学者阅读
