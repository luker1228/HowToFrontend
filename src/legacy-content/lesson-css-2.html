Figma 右侧属性面板，就是一行行 CSS。

你已经在 Figma 右侧面板填过 padding、改过圆角、调过字色。CSS 做的是同一件事，只不过把"面板字段"写成了"属性: 值"的文本。这一节用一个按钮，把两边对齐看清楚。

课程：CSS · Figma 视角 对比：Figma 属性 ↔ CSS 目标：看到属性名就知道写哪条 CSS

一句话核心

核心选中一个 Figma 图层，右侧那块属性面板，就是它"长什么样"的全部信息。CSS 把同一块信息写成了文本：一行一个属性，冒号后面跟值。字段名几乎一模一样。

一个按钮，两种描述

同一个按钮，左边是 Figma 属性面板，右边是浏览器看到的 CSS。鼠标移到任意一行，两边会同时高亮——它们是同一个东西。

Figma 视角
选中一个图层，右侧属性面板就是它"长什么样"的全部信息。
代码视角
一组 属性: 值。属性名几乎跟 Figma 一样：padding = 内边距、border-radius = 圆角。

把字段对上 CSS

  • Layout → 盒模型。Figma 的 padding、margin 区域，对应 CSS 的 padding、margin，单位也是 px。
  • Fill → 背景色。Figma 的 Fill，对应 background 或 background-color，颜色值直接抄十六进制。
  • Corner → 圆角。Figma 的 Corner radius，对应 border-radius。
  • Text → 文字。字号对应 font-size，字色对应 color。
  • 每个字段一行 CSS。Figma 把字段排成一块面板，CSS 把字段排成一段文本，本质是同一份清单。

属性面板和 CSS 都是"清单"

你去 Figma 右侧看一个图层，它写的是"这个图层有哪些字段、每个字段填了什么"。CSS 写的是同一件事，只是用文本列出来：background: #EF7627; 等于在 Fill 字段填了那个颜色。

所以读 CSS 的方式，跟读 Figma 属性面板一样：先认字段名，再看值。字段名你早就熟了——padding、radius、font-size——按钮这些都对应得上。

提示写 prompt 时与其说"按钮好看一点"，不如照着属性面板的清单说：按钮 padding 20px、圆角 8px、背景 #EF7627、字号 14px。给 AI 字段名，它不会乱猜。

小结

  • Figma 右侧属性面板和 CSS 是同一份清单，只是形态不同。
  • Layout → padding/margin，Fill → background，Corner → border-radius，Text → font-size/color。
  • 每个 Figma 字段对应一行 CSS，字段名几乎不用记新的。
  • 和 AI 协作时，照属性面板的清单写需求，比"好看一点"准得多。
