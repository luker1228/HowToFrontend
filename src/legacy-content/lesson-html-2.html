Figma 图层树，就是 HTML 标签树。

课程：HTML 第二课 对比：Figma ↔ HTML 目标：会读 DOM 结构

一句话核心

核心Figma 左侧的图层面板，和浏览器看到的 HTML，是同一棵树。你在设计稿里怎么嵌套 Frame，HTML 就怎么嵌套标签。记住这一点，看到任何设计稿都能估出大致结构。

设计视角
一棵图层树。Frame 套 Frame，每个图层有名字、有层级、有缩进。
代码视角
一棵标签树。<div> 套 <div>，每个标签有名字、有层级、有缩进——一模一样的事。

把规则说清楚

  • Frame = 标签。一个 Frame 对应一个容器标签，最常见的就是 <div>。
  • 缩进 = 嵌套。图层面板里缩进几格，HTML 里就嵌套几层。同级图层是兄弟，下层是子级。
  • 文字图层 = 文字标签。带 T 的图层，对应 <h1>~<h6>（标题）或 <p>（正文）。
  • 按钮 = <button>。不要用 <div> 当按钮——能点和不点是两件事。
  • 图层名 = class 名（大致）。设计师起的名往往就是你给标签起的 class，差不到哪去。

这棵树在浏览器里叫 DOM

Figma 叫它图层树，浏览器叫它 DOM（Document Object Model）。名字不同，结构一样：根节点套子节点，子节点再套孙节点，一路展开。

为什么前端总提 DOM？因为后续所有事——改样式（CSS）、加交互（JS）、取数据填进去——都是在「找准某个节点，对它动手」。你会读图层树，就会读 DOM；会拆 Frame，就会拆标签。

提示写 prompt 时与其说「这里加个块」，不如说「在 Card 这层下、Header 旁边加一个新 <div>」。给 AI 的也是一棵树的位置，AI 才不会插错地方。

小结

  • Figma 图层面板和 HTML 是同一棵树，只是名字不同。
  • Frame → <div>，文字图层 → <h1>/<p>，按钮 → <button>。
  • 缩进就是嵌套，同级就是兄弟。
  • 浏览器里这棵树叫 DOM，所有前端活都在它上面做。
